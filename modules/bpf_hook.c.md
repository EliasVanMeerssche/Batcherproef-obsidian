**Verbergen van de Command & Control (C2) verbinding:** De rootkit communiceert met een externe server (`YOUR_SRV_IP`) over poort 8081 (`HIDDEN_PORT`). In plaats van netwerkverkeer op laag niveau te droppen via bijvoorbeeld XDP, kaapt deze module de functie `bpf_iter_run_prog`. Deze functie wordt vaak door monitoringtools gebruikt om actieve netwerksockets uit te lezen. De functie `should_hide_socket_port` zorgt ervoor dat, als een socket poort 8081 of het IP-adres van de aanvaller gebruikt, deze simpelweg wordt overgeslagen en uit de output wordt gefilterd. Hierdoor is de C2-verbinding volledig onzichtbaar voor netwerkinspectietools.

**Gerichte uitschakeling van detectiesoftware (Falco):** Het meest opvallende aan deze module is dat het specifieke kanalen blokkeert waarmee eBPF alarmen naar userspace stuurt. De code bevat een expliciete functie genaamd `is_falco_event` die datapakketten controleert op de `falco_event_hdr`-structuur. Door hooks te plaatsen op ringbuffers (`bpf_ringbuf_output`, `bpf_ringbuf_submit`) en perf events (`perf_event_output`), zorgt de rootkit ervoor dat events of waarschuwingen die door de malware worden gegenereerd geruisloos worden gedropt en Falco nooit bereiken.
